01

Words You'll Hear

Before anything else — learn the language. These are the terms that pop up daily on every WordPress project. If you don't know them yet, this is where you start.

Site environments

A website doesn't just exist in one place. It usually has three "versions":

Live Site
The real, public website that visitors see. Also called the production site.
Staging Site
A private copy of the live site used for testing before you make changes public.
Dev Site
A draft version where new features and designs are built first, before going to staging.
Simple analogy Think of it like cooking. Dev is your test kitchen, Staging is the dish plated for tasting, and Live is what's served to the customer.

Safety & maintenance

Backup
A saved copy of the website you can restore if something breaks.
Restore
The act of bringing the site back from a backup.
Update
Installing the newest version of WordPress core, a theme, or a plugin to stay secure.
Migration
Moving a website from one server or domain to another.
Downtime
When the site is offline or unreachable.
Rollback
Undoing a recent update or change.
Deployment
Pushing changes from staging or dev up to the live site.

Website foundation

Domain
The website's address, like example.com.
Hosting
The server (computer) where your website files actually live.
DNS
The "map" that connects your domain to your hosting.
Subdomain
An extra address under your domain, like blog.example.com.
SSL Certificate
What makes your site secure — the padlock icon in the browser.

WordPress building blocks

Theme
The design template of your site.
Plugin
An add-on app that adds extra features (forms, SEO, security, etc.).
Page
Static content like Home, About, or Contact.
Post
Blog articles or news updates.
Media Library
Where all your uploaded images, videos, and documents are stored.
Menu
The clickable links at the top, side, or bottom that move you around the site.
Widget
Small add-ons placed in sidebars or footers (search bars, calendars, etc.).
Shortcode
A small code snippet you paste into a page to display something (like a form or gallery).
Cache
A "saved snapshot" of the site that loads faster for returning visitors.

Design parts

Header
The top section of the website (logo + main menu).
Footer
The bottom section (contact info, copyright, secondary links).
Sidebar
The narrow side area where widgets can sit.
Customizer
WordPress's built-in tool for adjusting colors, logo, and fonts.

User roles (who can do what)

Admin
Full control. Can do anything, including delete the site.
Editor
Can manage and publish all posts and pages.
Author
Can write and publish their own posts only.
Contributor
Can write posts but can't publish them.
Subscriber
Can only read and log in. No editing.

If the site sells products (WooCommerce)

Cart
Where products go before checkout.
Checkout
The page where customers pay.
SKU
A unique product code or ID.
Inventory
Stock management for products.
Gateway
The payment method (PayPal, Stripe, etc.).

Performance & security

CDN
Content Delivery Network — speeds up your site worldwide by serving files from the nearest location.
Firewall
Security barrier that blocks hackers.
Malware Scan
Checking the site for viruses or harmful code.
Uptime
How long the site stays online without going down.

Best Approach to Designing a Website

Don't open Figma yet. Don't pick colors yet. Great websites start with thinking, not designing. Follow these five steps in order.

STEP 1 Understand the Goal

Before you draw a single box, answer these three questions:

  • Who is the target audience? Who are we building this for?
  • What is the main purpose? Sales, info, bookings, leads?
  • What action should users take? This is the CTA (Call To Action) — buy, call, sign up, download.
STEP 2 Plan the Structure (UX)
  • Create a site map — a list of every page the site will have.
  • Sketch a wireframe — a rough layout of each page (Home, About, Services, etc.).
  • Plan the user flow — how a visitor moves from one page to the next.
STEP 3 Design the UI (User Interface)
  • Choose a clean, consistent design system: colors, fonts, spacing.
  • Use a grid layout and proper hierarchy (H1 → H2 → H3).
  • Design for both desktop and mobile (this is what "responsive" means).
  • Work section-by-section: finish one section completely (desktop → mobile) before moving to the next. See Chapter 07 for the full workflow.
STEP 4 Use Real Content Early
  • Avoid lorem ipsum and dummy text. Real content reveals real layout problems.
  • Optimize images, icons, and graphics. Use TinyPNG for images and HandBrake for video.
STEP 5 Focus on Conversion
  • Add clear CTAs — buttons, forms, contact info that stand out.
  • Build trust with testimonials, certifications, real photos.
  • Avoid clutter. Whitespace is your friend.
Why this order matters Skipping straight to design is the #1 mistake beginners make. You'll spend weeks making something pretty that doesn't actually solve the client's business problem.

03

Structuring a Website

After the design is done, here's everything you need to set up so the site actually works in the real world.

📄 On-Page SEO

  • Set meta titles and meta descriptions on every page (with relevant keywords).
  • Use only one H1 per page, then structure with H2 and H3.
  • Use internal links (linking to your own pages) and outbound links (to trusted sources).

⚡ Performance Optimization

  • Compress images using WebP format or TinyPNG.
  • Enable caching and minify your CSS and JavaScript.
  • Use a CDN like Cloudflare for global speed.

📱 Mobile Responsiveness

  • Make sure the site works on every screen size.
  • Test using Chrome DevTools or a mobile emulator.

🔐 Security & Backup

  • Install SSL (HTTPS) — without this, browsers show "Not Secure".
  • Set up regular automatic backups.
  • Use a security plugin like Wordfence.

📈 Analytics & Tracking

  • Install Google Analytics so you can see visitor data.
  • Set up Google Search Console.
  • Submit your sitemap.xml to Google.

SEO Heading Tags

Headings (H1, H2, H3) aren't just about size — they tell Google what your page is about. Used right, they boost your ranking. Used wrong, they hurt it.

The rule

  • H1: Only one per page. This is the main topic or title of the page.
  • H2: Use for main sections under the H1. You can have multiple.
  • H3: Use for sub-sections under H2s. Also multiple allowed.
Example structure H1: Plumbing Services in Los Angeles
  H2: Emergency Plumbing
    H3: 24/7 Pipe Repair
    H3: Water Heater Replacement
  H2: Routine Maintenance
  H2: Why Choose Us

Why this matters

  • Helps Google understand your page structure.
  • Improves readability for users.
  • Supports keyword relevance when used properly.

05

Header & Footer Standards

The header and footer appear on every page of the website. Get them right once, and the whole site benefits.

Standard Header Design

  • Logo at top-left — make sure it's clickable and links to the homepage.
  • Navigation Menu — align right or center. Keep it to 5–7 links max.
  • Call-to-Action button — separate from the menu. Examples: "Contact Us", "Shop Now", "Get a Quote".
  • Sticky/Fixed — make it stick to the top so it's visible even when scrolling down.
  • Responsive Setup — desktop shows full menu; mobile shows a hamburger menu (☰).

Standard Footer Design

  • 4-column layout on desktop: About | Quick Links | Services/Products | Contact Info.
  • Copyright + small menu at the very bottom, centered.
  • Social media icons — keep them simple and consistent (right side or centered).
  • Newsletter signup — optional, but great for engagement.
  • Mobile layout — columns automatically stack vertically.

Pro tips so it doesn't break later

  • Use Global Templates in WordPress (Elementor, Divi, or Kadence). Build the header/footer once as a template, then apply it to all pages. Edit one place, update everywhere.
  • Use Flexbox or Grid for layout. Avoid absolute positioning — it breaks easily when the screen resizes.
  • Be consistent with spacing. Set a padding/margin scale (example: 20px on desktop, 10px on mobile).
  • Save reusable blocks so you can copy-paste them into other projects.

06

Padding & Spacing Units

When to use px, %, em, or rem. Choosing the wrong one is why so many sites break on mobile.

1. Pixels (px)

Fixed size — does not adjust on different screens.

  • Example: padding: 20px;
  • Best for: small details that should stay the same — icons, buttons, logos.
Watch out If you use px for everything, your layout will break on mobile because pixels don't adapt to screen size.

2. Percentage (%)

Relative to the parent container — adjusts based on the section/column width.

  • Example: padding: 5%; — smaller screen means smaller padding automatically.
  • Best for: responsive sections that need to stay proportional.
  • Advantage: you don't have to write separate rules for each screen size.

3. em and rem

  • em = relative to the font size of the parent element.
  • rem = relative to the root font size (the global one).
  • Example: padding: 2rem; — if your base font is 16px, padding becomes 32px.
  • Best for: consistent spacing tied to text size, so everything feels uniform.

The JC simple rule

Memorize this one rule Left & Right padding = 20px on every section, desktop and mobile. Always. No exceptions.

Why? Easy to remember, easy to apply, and any section you build is instantly copyable to the next project — the spacing is already correct.

Once L/R is locked at 20px, you only need to think about top and bottom padding (which changes between desktop and mobile). Everything else stays the same.

Other quick rules

  • Use px for small UI elements: buttons, icons, logos.
  • Use rem for spacing tied to text (column inner padding).
  • Stick to the 8px spacing scale: 8, 16, 24, 32, 40, 48 — multiples of 8 only.
  • Set spacing globally in your theme/builder so one change updates the whole site.
JC Standard Padding Section L/R padding: 20px (desktop & mobile — same!)
Section top-bottom: 80–120px desktop / 40–60px mobile
Column inner padding: 2rem
Button padding: 12px 24px

07

Section-by-Section Workflow

The fastest way to build a site: finish one section completely (desktop + mobile) before moving to the next. With L/R padding locked at 20px, every section becomes copy-paste ready.

The Golden Rule

JC Padding Standard Left & Right = 20px on every section — desktop and mobile, same value. Easy to remember, easy to copy.

Only top & bottom changes between desktop and mobile.

The Workflow

  1. Pick one section (e.g., Hero).
  2. Build the desktop version with the standard padding below.
  3. Switch to mobile view and adjust top/bottom padding (L/R stays 20px).
  4. Save the section as a reusable block.
  5. Move to the next section and repeat.

Because every section uses the same 20px L/R padding, you can copy any section to a new project and it will already have the correct spacing.

Standard Section Padding

SECTION Hero / Main Blocks

Desktop (build first)

  • Top & Bottom: 80px – 120px
  • Left & Right: 20px

Mobile (do this next)

  • Top & Bottom: 40px – 60px
  • Left & Right: 20px ← same as desktop
Real Hero example Desktop: padding: 100px 20px;
Mobile: padding: 50px 20px;
SECTION Columns / Inner Content

Desktop (build first)

  • Inner padding: 40px
  • Gap between columns: 32px – 48px

Mobile (do this next)

  • Inner padding: 16px / 20px / 24px
  • Columns auto-stack into one column
SECTION Text Blocks

Desktop (build first)

  • Paragraph line-height: 1.5rem
  • Gap between sections: 64px – 96px

Mobile (do this next)

  • Paragraph line-height: 1.2rem
  • Gap between sections: 32px – 48px

Quick Reference Table

Section Desktop top/bottom Mobile top/bottom L/R (both)
Hero 80–120px 40–60px 20px
Standard sections 80px 40px 20px
Compact sections 64px 32px 20px

Pro tips

  • Save sections as reusable blocks in your builder (Elementor, Divi, Kadence). One section, infinite reuse. See Chapter 14 for the full how-to.
  • Stick to the 8px scale for top/bottom values: 8, 16, 24, 32, 40, 48, 64, 80, 96, 120.
  • Test in 3 views: Desktop, Tablet, Mobile. Tablet often breaks first.
  • Don't change L/R padding per section. If you do, your site loses its consistent rhythm.

Image Sizes (Web Standard)

Right size, right format, right alt text. This is the difference between a fast site and a slow one.

Why standard image sizes matter

  • Faster page load — critical for SEO and mobile users.
  • Consistent design across devices.
  • Prevents layout shifts and improves Core Web Vitals.
  • Pages look clean and professional.

Standard Image Sizes

Image Type Size (W × H) Max File Size Notes
Logo250 × 100≤ 50 KBSVG or PNG preferred
Hero Banner1920 × 700–800≤ 250–400 KBJPG or WebP for performance
Inner Banner1920 × 500–600≤ 200–300 KBFor section headers
Content Image800 × 600≤ 100 KBInside blogs or pages
Full-Width Image1920 × 1080≤ 250–350 KBBackground or featured
Mobile Banner768 × 1000≤ 150–200 KBOptimize separately
Thumbnail150 × 150≤ 30–50 KBGalleries or cards

Filename Guidelines

  • Use descriptive, keyword-rich names.
  • All lowercase, with hyphens (-) between words.
  • No special characters or spaces.
✓ Do plumbing-services-los-angeles.jpg
modern-office-interior-design.webp
✗ Don't IMG_4837.JPG
Picture 1.png
image_final_FINAL.jpg

Alt Text Guidelines

  • Describe the image briefly and clearly.
  • Include keywords naturally — but don't stuff.
  • Avoid generic terms like "image of" or "picture of".
Example Filename: kitchen-renovation-austin.jpg
Alt Text: "Kitchen renovation in Austin with white marble countertops"

Image Naming & Format Guide

A naming convention so consistent that any teammate can find any file in seconds, and a format guide so you never use the wrong file type again.

The naming formula

page-section-image-role[-context-or-variant][-size-or-resolution].ext

Rules

  • Use only hyphens (-). Never underscores.
  • All lowercase.
  • Be descriptive and consistent.

What each part means

PartDescription
pageThe page the image is used on (home, services, blog...)
sectionThe section within that page (hero, gallery, footer...)
image-roleWhat the image is (hero, icon, photo, mockup...)
variant (optional)Theme or interaction state (dark, hover, open...)
size (optional)Display variant (@2x, mobile, thumb...)
extFile extension (.svg, .webp, .jpg, .png)

Real examples

home-hero-background.webp
services-icon-consulting-dark.svg
product-growkit-mockup-front@2x.webp
blog-how-to-grow-traffic-feature-image.webp
case-study-harvestplus-overview-photo.jpg
header-logo-light.svg
footer-logo-dark.svg
home-gallery-carousel-02.webp
faq-accordion-open-icon.svg

When to use which file format

SVG (.svg) — for icons, logos, flat illustrations

✓ Use when You need responsive scaling. You need crisp UI at any size. You need transparent icons or logos.
✗ Avoid when The image is a photo. There's heavy graphical detail. Complex bitmap effects are involved.

WebP (.webp) — for photos, UI graphics, backgrounds

✓ Use when You want high quality with a small file size. You need transparency. You're optimizing carousels and galleries.
✗ Avoid when You need it for print. Your editing workflow needs PSD or doesn't support WebP.

JPEG (.jpg) — for high-res photos, fallback or editing base

✓ Use when You need photographic content. You need a WebP fallback. The image is heading to heavy editing.
✗ Avoid when You need transparency. The image is a vector or illustration.

PNG (.png) — for transparency when SVG/WebP don't work

✓ Use when You need a UI overlay with transparency. You need image masks.
✗ Avoid when It's a hero banner or full-width background. It's an icon or logo (use SVG instead).

WordPress Review & Testing Process

The full pre-launch checklist. Run through this in order, every time, before any site goes live. No skipping.

PHASE 1 Theme Cleanup & Content Review

Do this first — it catches the most common issues.

1.1 — Remove demo content & theme elements

  • Header/Navigation: Remove sample menu items like "Sample Page" or "Hello World". Go to Appearance → Menus.
  • Footer: Remove theme credits and sample text. Check Appearance → Customize → Footer for "Powered by [Theme Name]".
  • Widgets: Remove default widgets like "Recent Posts" or "Sample Widget" via Appearance → Widgets.
  • Homepage: Search for lorem ipsum, "Your content here", or stock theme images. Replace with the client's real content.
  • Blog/News: Delete the "Hello World" post and any other demo posts. If no blog is planned, hide the menu item.

1.2 — Elementor-specific cleanup

  • Check for Elementor demo templates still active.
  • Remove unused Elementor template library items.
  • Verify all Elementor widgets show client content, not placeholders.
PHASE 2 Functionality Testing

Test that every interactive element actually works.

2.1 — Forms

  • Fill out and submit every form on the site.
  • Check that email notifications actually arrive in the inbox.
  • Make sure confirmation messages display after submission.
  • Test required field validation (try submitting empty fields).
  • For quote forms: verify all logic, calculations, and dropdowns work.

2.2 — Navigation & links

  • Main menu: Click every link. Confirm submenus open.
  • CTA buttons: "Call Now" should open the phone dialer; "Email Us" should open the email client.
  • Internal links: Click every "Learn More" and inline link to confirm they go where they should.
PHASE 3 Cross-Browser & Device Testing

3.1 — Browsers

Test in (at minimum):

  • Chrome (desktop)
  • Firefox (desktop)
  • Safari (if you have a Mac)
  • Edge (Windows)

In each: confirm pages load fully, layout is correct, buttons and forms work, images display properly.

3.2 — Mobile

  • Test on your actual phone (this is the easiest method).
  • Use Chrome's mobile view: press F12 → click the mobile icon → test iPhone, iPad, Android sizes.
  • Confirm: hamburger menu works, text is readable without zooming, buttons are easy to tap, forms work, images resize.
PHASE 4 Performance & Speed Testing

4.1 — Speed tests

  • Google PageSpeed Insights: go to pagespeed.web.dev, paste the URL. Target green (80+) on both desktop and mobile.
  • GTmetrix: target Grade A or B, with load time under 3 seconds.

4.2 — Image optimization

  • Find any images over 1MB and compress them (use TinyPNG).
  • Resize images down to actual display size — don't serve a 4000px image to a 400px slot.
  • Verify every image has alt text. Right-click → Inspect → check for the alt="" attribute.
PHASE 5 Content & SEO Basics

5.1 — Content review

  • Read through main pages for spelling and grammar (Grammarly helps).
  • Verify all phone numbers are correct and clickable on mobile.
  • Confirm address matches Google My Business.
  • Check email addresses actually work.
  • Make sure business hours are current.

5.2 — Basic SEO

  • Page titles in browser tabs should be descriptive — never "Home" or "Page".
  • Install Rank Math (or similar SEO plugin) if not already installed.
  • Add meta descriptions to homepage, services, and contact (under 160 characters each).
PHASE 6 Error Detection & Resolution

6.1 — Broken links

  • Manually click all major links.
  • Open browser console (F12) and look for red error messages.
  • Test 404 errors by visiting a URL that doesn't exist — confirm a proper 404 page shows.

6.2 — Visual issues

  • Look for broken image icons.
  • Check for overlapping text or images.
  • Verify text contrast — is everything readable?
  • Watch for cramped or overly spread-out content.
PHASE 7 Final Pre-Launch Checks

7.1 — WordPress settings

  • Search Engine Visibility: Go to Settings → Reading. UNCHECK "Discourage search engines". This is the #1 launch mistake!
  • Permalinks: Go to Settings → Permalinks. Choose "Post name" structure for clean URLs.

7.2 — Security & backup

  • Remove demo admin accounts.
  • Update all plugins to their latest versions.
  • Create a fresh backup before launch.
PHASE 8 Document Issues & Handoff

8.1 — Document any issues found

  • Take screenshots.
  • Note the page URL where the issue happens.
  • Describe what's wrong in plain language.
  • Fix what you can; flag the rest for the SEO specialist.

8.2 — Pre-SEO handoff checklist

Before passing to the SEO specialist, confirm:

  • All demo content removed
  • All forms work and emails arrive
  • Site loads in under 3 seconds
  • Mobile version looks good
  • Contact info is accurate
  • Search engines are allowed, not blocked

Training Phase Guide

A structured 1-month plan to take a complete beginner from zero to ready-to-ship a real WordPress site. Hands-on tasks every day, checkpoints every week, and a final project that proves it.

How this works

  • 4 weeks, each focused on one phase of the workflow.
  • ~2 hours/day of focused practice, 5 days a week.
  • End each week with a checkpoint. Don't move on until you can check every box.
  • Week 4 ends with a final project — a complete dummy client site you build solo.
Why one month Web design isn't memorization — it's muscle memory. One month gives you enough time to learn each concept, practice it repeatedly, and start making it second nature. Rushing leads to gaps that show up later in client work.

Elementor Globals & Reusable Templates

Build it once, use it everywhere. Update it once, it changes everywhere. This is the secret to fast, consistent, low-maintenance websites — and the cure for the "I changed the button color on 47 pages" nightmare.

Why this matters

Without globals and templates, every section, button, and color is a one-off. Want to change the brand color from blue to green? You're editing every page manually. With globals, you change it once in one place — every page updates automatically. Same for spacing, fonts, headers, footers, CTAs, testimonials, and any block you reuse.

The mental model Think of globals as your "design DNA" and templates as your "Lego blocks." Set the DNA once. Build with the blocks forever.

Part 1 — Global Site Settings

In Elementor, "Site Settings" is where you define your design system once. Open Elementor, click the hamburger menu (☰) at the top-left, and choose Site Settings.

Global Colors

Set up 4–6 brand colors that you'll use across the entire site. When you change one here, every element using that color updates instantly.

  • Primary — your main brand color (used for buttons, links, accents)
  • Secondary — supporting brand color
  • Text — main body text color (usually a dark gray, not pure black)
  • Accent — for CTAs, highlights, hover states
  • Background — section backgrounds (often white or very light gray)
Important Always pick colors from the global palette when styling elements. Never type a hex code directly into a widget — that breaks the global system.

Global Fonts

Set up your typography exactly like Chapter 8 specifies. In Site Settings → Typography:

  • Primary — for headings (Title 1–7)
  • Secondary — for body text (Body L, Body, Body S)
  • Text — fallback for general text
  • Accent — for special emphasis (quotes, eyebrows, callouts)

Theme Style

Site Settings → Theme Style is where you set the defaults for every basic element on the site:

  • Typography — H1 through H6 default sizes (use the JC scale from Chapter 8)
  • Buttons — default padding (12px 24px), border-radius, hover state
  • Form fields — input borders, padding, focus colors
  • Images — default border-radius if any

Layout Settings

  • Container width: 1200px or 1240px (standard)
  • Widgets space: 0 (manage spacing per section)
  • Page Layout: Elementor Canvas or Full Width — depends on your build
JC Setup Order When starting any new site: (1) Site Settings → Global Colors first. (2) Then Global Fonts. (3) Then Theme Style. (4) Then Layout Settings. Do this BEFORE building any pages.

Part 2 — Reusable Section Templates

Once your section is built (desktop + mobile per Chapter 7), save it. There are three ways to reuse content in Elementor — each with a specific purpose.

Method 1 — Saved Templates (most common)

Save any section, container, or page as a template you can drop into other pages.

  1. Right-click the section/container you want to save.
  2. Click Save as Template.
  3. Give it a clear, consistent name (see naming rules below).
  4. To reuse: open another page, click the folder icon (Templates), find your saved template, and insert.
Note Saved Templates are copies. If you edit the template, the existing inserted instances on other pages do NOT update. Use Method 3 if you want true synced content.

Method 2 — Global Widgets (Pro only)

Save a single widget (button, heading, form) globally. When you edit it in one place, every instance updates automatically across the site.

  1. Right-click any widget → Save as Global.
  2. Drag the global widget from the panel into other pages.
  3. Edit it once → updates everywhere it appears.

Best for: CTA buttons, contact info blocks, social icons, footer phone numbers — anything that must stay identical site-wide.

Method 3 — Theme Builder (for header, footer, archives)

For elements that appear on every page (like header and footer), use Elementor's Theme Builder. This is the right tool for site-wide structural pieces.

  1. Go to Templates → Theme Builder.
  2. Choose the part: Header, Footer, Single Post, Archive, 404, etc.
  3. Build it once, set display conditions to "Entire Site" (or specific pages).
  4. Edit anytime — changes appear instantly on every page where it's displayed.
Real workflow Header & footer → Theme Builder. Site-wide CTA button → Global Widget. Hero/Services/Testimonials sections → Saved Templates. That's the right tool for each job.

When to use which method

If you're saving... Use this method Why
Header / Footer Theme Builder Appears on every page, must update site-wide
Hero, About, Services, Testimonials sections Saved Templates Reused across pages but edited per page
"Get a Quote" CTA button Global Widget Must stay identical and update everywhere
Phone number / contact info Global Widget If it changes, you don't want to miss any page
Single blog post / product layout Theme Builder (Single) All blog posts share the same structure
FAQ block, pricing table Saved Template Reusable building block, edited per use

Template Naming Convention

Just like image naming (Chapter 10), give your templates clear names so you (or anyone else) can find them in 2 seconds.

The formula

[client]-[type]-[section]-[variant]

Examples

acmeplumbing-section-hero-home
acmeplumbing-section-services-grid
acmeplumbing-section-testimonials-slider
acmeplumbing-section-cta-newsletter
acmeplumbing-block-faq-pricing
acmeplumbing-global-button-primary
acmeplumbing-global-contactinfo-footer

Type prefixes

  • section- — full-width section (Hero, Services, About)
  • block- — smaller reusable component (FAQ, pricing card, testimonial)
  • global- — global widget (button, contact info, social icons)
  • page- — full page template

The full workflow (every new project)

STEP 1 Set up globals first
  • Open Site Settings.
  • Define Global Colors (Primary, Secondary, Text, Accent, Background).
  • Define Global Fonts (Primary headings + Secondary body).
  • Configure Theme Style (H1-H6 sizes, button defaults, form fields).
  • Set container width and layout defaults.
STEP 2 Build header and footer in Theme Builder
  • Templates → Theme Builder → Header.
  • Build per Chapter 5 standards (logo, 5–7 menu items, CTA, sticky).
  • Set display: "Entire Site". Publish.
  • Repeat for Footer (4-column desktop, stacked mobile).
STEP 3 Build sections one at a time
  • Build Hero section (desktop → mobile, 20px L/R per Chapter 7).
  • Right-click → Save as Template using the JC naming format.
  • Move to next section (About, Services, etc). Repeat.
STEP 4 Save universal CTAs as Global Widgets
  • Build the main CTA button once.
  • Right-click → Save as Global.
  • Now use it on every page — one edit updates all instances.
STEP 5 Assemble pages from your library
  • Open a new page.
  • Insert templates one by one in the right order.
  • Tweak content (text, images) but never the structure or spacing.
  • Preview, save, move on.

Common mistakes to avoid

✓ Do
  • Set up Site Settings before building any pages
  • Use Global Colors from the palette, never raw hex codes
  • Build header/footer in Theme Builder, not as page sections
  • Name templates with the JC format every time
  • Test edits on one page before pushing to global
✗ Don't
  • Hard-code colors and fonts directly into widgets
  • Copy-paste the header onto every page manually
  • Save 50 templates with names like "section1", "test", "final-final"
  • Edit a Saved Template expecting all inserted copies to update (they won't)
  • Skip the Theme Style setup and style each H1 manually

The bottom line A well-set-up Elementor site can be re-themed for a new brand color in under 5 minutes. A poorly set-up one takes 5 hours. The difference is everything in this chapter.
